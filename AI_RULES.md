# AI_RULES.md: Стандарты и правила написания кода на 1С

Данный документ описывает стандарты кодирования для ИИ-ассистентов при генерации, рефакторинге и дополнении кода на языке 1С:Предприятие.

## 0. Технические требования и среда

*   **Платформа:** 1С:Предприятие версии 8.3.26 и выше.
*   **Режим работы:** Управляемые формы.
*   **Язык и типизация:** Язык 1С (BSL).

## 1. Именование переменных и процедур

### 1.1. Переменные
*   **Стиль:** CamelCase или PascalCase. Имена должны быть осмысленными и отражать суть данных.
*   **Префикс `Тек`**: используется для переменных внутри циклов или обозначающих «текущий» объект/значение.
    *   *Примеры:* `ТекСтадии`, `ТекДанные`, `ТекВыбранныеФайлы`, `ТекМассивОтсутствие`.
*   **Суффикс `Вход`**: используется для входных параметров процедур и функций, чтобы отличать их от локальных переменных.
    *   *Примеры:* `ИнициативаВход`, `ЗадачаВход`, `ИдентификаторТекущейСтадииВход`.
    *   **Неизменяемость:** параметры с суффиксом `Вход` запрещено изменять внутри метода. Если требуется модифицировать значение, необходимо создать новую локальную переменную.
        *   *Правильно:*
            ```bsl
            Процедура ОбработатьЗначение(ЗначениеВход)
                ЗначениеРезультат = ЗначениеВход + 1;
            КонецПроцедуры
            ```
        *   *Неправильно:*
            ```bsl
            Процедура ОбработатьЗначение(ЗначениеВход)
                ЗначениеВход = ЗначениеВход + 1;
            КонецПроцедуры
            ```
*   **Суффикс `Изм`**: используется для параметров, которые метод намеренно изменяет (например, заполняет переданную структуру или массив). Суффикс указывает вызывающему коду, что параметр будет модифицирован.
    *   *Примеры:* `СтруктураИзм`, `МассивРезультатовИзм`, `ОбъектИзм`.
    *   *Пример использования:*
        ```bsl
        Процедура ЗаполнитьСтруктуруДанными(ИсточникДанных, СтруктураИзм)
            СтруктураИзм.Вставить("Ключ1", ИсточникДанных.Ключ1);
            СтруктураИзм.Вставить("Ключ2", ИсточникДанных.Ключ2);
        КонецПроцедуры
        ```
*   **Булевы переменные:** начинаются с `Это`, `Есть` или `Разрешено`.
    *   *Примеры:* `ЭтоВысокийПриоритет`, `ЕстьВводОшибки`, `РазрешеноНапоминаниеОПаузе`.
*   **Промежуточные результаты:** в функциях используется переменная `РезультатФункции` для формирования возвращаемого значения перед оператором `Возврат`.

### 1.2. Процедры и Функции
*   **Стиль:** PascalCase.
*   **Формат:** `Глагол` + `Существительное` (например, `ПолучитьТекстИсторииИнициативы`, `РассчитатьОбщиеТрудозатраты`, `УправлениеДоступностью`).
*   **Обработчики событий:** именование соответствует стандарту 1С (например, `СтадииПриИзменении`, `ПриСозданииНаСервере`, `ВзаимодействияСтадийПриНачалеРедактирования`).

## 2. Клиент-серверное взаимодействие и контекст

*   **Директивы компиляции:** для каждого метода явно указывается директива:
    *   `&НаКлиенте` — для методов, работающих с интерфейсом и событиями формы.
    *   `&НаСервере` — для методов, требующих контекста сервера.
    *   `&НаСервереБезКонтекста` — для тяжёлых вычислений, запросов к БД и бизнес-логики (предпочтительный вариант для серверных функций).
    *   `&НаКлиентеНаСервереБезКонтекста` — для универсальных методов, не зависящих от контекста (например, `УправлениеДоступностью(Форма)`).
*   **Передача формы:** при вызове универсальных процедур на клиенте/сервере без контекста форма передаётся явным параметром (например, `УправлениеДоступностью(ЭтаФорма)` или `СчитатьИсториюИнициативы(Форма)`).
*   **Минимизация контекста:** передача всего объекта на клиент не допускается, если можно передать только необходимые данные.

## 3. Структура кода, ветвление и форматирование

### 3.1. Ветвление
*   **Обработка всех веток:** условие `Если ... Тогда` имеет блок `Иначе`. Если в блоке `Иначе` не требуется действий, он заполняется комментарием, поясняющим отсутствие действий.
    *   *Правильно:*
        ```bsl
        Если ЗначениеЗаполнено(Объект.Ссылка) Тогда
            // ... полезный код ...
        Иначе
            // Пользователь отказался. Ничего не делаем.
        КонецЕсли;
        ```
    *   *Неправильно:* оставлять `Иначе` пустым или опускать его.

### 3.2. Форматирование и пустые строки
*   **Пустые строки только между методами:** внутри тела метода (процедуры или функции) пустые строки для разделения логических блоков не используются. Пустая строка допускается только между методами (между `КонецПроцедуры` / `КонецФункции` и следующим комментарием или директивой следующего метода).
*   **Логическое разделение внутри метода:** вместо пустых строк для разделения блоков внутри метода используются комментарии-заголовки (например, `// Инициализация.`, `// Заполнение нового.`, `// Обновление состояние из регистра.`).

### 3.3. Принцип «Одна строка — одно действие»
*   **Атомарность строк:** каждая строка кода выполняет одно действие. Это обеспечивает удобство отладки и пошагового выполнения.
*   **Запрет на составные выражения:** в одной строке не объединяются несколько операций, присваиваний или вызовов функций.
    *   *Правильно:*
        ```bsl
        РезультатЗапроса = Запрос.Выполнить();
        РезультатФункции = НЕ РезультатЗапроса.Пустой();
        Возврат РезультатФункции;
        ```
    *   *Неправильно:*
        ```bsl
        Возврат НЕ Запрос.Выполнить().Пустой();
        ```
*   **Исключения:** допускается простая инициализация переменных примитивными значениями в одну строку, если это не затрудняет чтение.

## 4. Структура и размер методов

### 4.1. Ограничение длины метода
*   **Правило одного-двух экранов:** тело метода помещается на один экран (стандартное разрешение монитора). В исключительных случаях допускается максимум два экрана.
*   **Рефакторинг длинных методов:** если метод превышает указанный размер, его необходимо разбить на несколько подчинённых процедур или функций с осмысленными именами, отражающими суть выполняемых действий.

### 4.2. Секция инициализации
*   **Инициализация в начале метода:** все предопределённые значения, константы, состояния и ссылки на перечисления, используемые в методе, описываются в начале метода в отдельной секции с комментарием `// Инициализация.`.
*   **Пример:**
    ```bsl
    &НаСервере
    Процедура ПриСозданииНаСервере(Отказ, СтандартнаяОбработка)
        // Инициализация.
        СостояниеЧерновик = Перечисления.СостоянияЗадач.Черновик;
        СтрокаЗапускаБраузера = Константы.СтрокаЗапускаБраузера.Получить();
        СостояниеЗавершена = ПредопределенноеЗначение("Перечисление.СостоянияЗадач.Завершена");
        ВидСитуацииОшибка = ПредопределенноеЗначение("Справочник.ВидыСитуаций.Ошибка");
        // Заполнение нового.
        // ... остальной код ...
    КонецПроцедуры
    ```

## 5. Комментирование

*   **Документирующие комментарии:** для функций и процедур, используемых в других модулях или выполняющих сложную логику, используется стандартный блок 1С:
    ```bsl
    // Получает текст истории инициативы.
    //
    // Параметры:
    //  ИнициативаВход - ДокументСсылка.Инициатива - инициатива, историю которой требуется получить.
    //
    // Возвращаемое значение:
    //  Строка - текст истории инициативы.
    //
    ```
*   **Внутренние комментарии:** пишутся на русском языке, поясняют выполняемые блоком кода действия или принятые решения. Размещаются внутри блоков кода.
*   **Завершение методов:** допускается комментарий в конце сложной функции с её именем, например: `КонецФункции // ПроверитьНаличиеВводаОшибки()`.

## 6. Работа с данными, коллекциями и запросами

### 6.1. Проверки и структуры
*   **Проверка заполнения:** для проверки ссылок, дат, строк и чисел на пустоту используется `ЗначениеЗаполнено(...)`.
*   **Проверка объектов:** для проверки структур, массивов и объектов на наличие используется сравнение с `Неопределено`.
*   **Создание структур:** предпочтительно явное создание и заполнение структуры через `Вставить`, а не конструктор со строкой ключей.
    *   *Предпочтительно:*
        ```bsl
        СтруктураПараметров = Новый Структура;
        СтруктураПараметров.Вставить("ИмяФайла", ТекВыбранныеФайлы);
        СтруктураПараметров.Вставить("СсылкаНаОбъект", Объект.Ссылка);
        ```

### 6.2. Запросы и производительность
*   **Запрет вызовов БД в цикле:** выполнение запросов к базе данных внутри циклов запрещено. Это снижает производительность.
    *   *Неправильно:*
        ```bsl
        Для Каждого Элемент Из МассивЭлементов Цикл
            Запрос = Новый Запрос;
            Запрос.Текст = "ВЫБРАТЬ ... ГДЕ Ссылка = &Ссылка";
            Запрос.УстановитьПараметр("Ссылка", Элемент.Ссылка);
            Результат = Запрос.Выполнить();
        КонецЦикла;
        ```
*   **Пакетная обработка:** вместо циклов с запросами используется пакетная обработка — массив значений передаётся в параметр запроса, все данные обрабатываются одним запросом.
    *   *Правильно:*
        ```bsl
        // Формируем массив ссылок
        МассивСсылок = Новый Массив;
        Для Каждого Элемент Из МассивЭлементов Цикл
            МассивСсылок.Добавить(Элемент.Ссылка);
        КонецЦикла;
        // Один запрос на все данные
        Запрос = Новый Запрос;
        Запрос.Текст = "ВЫБРАТЬ ... ГДЕ Ссылка В (&МассивСсылок)";
        Запрос.УстановитьПараметр("МассивСсылок", МассивСсылок);
        РезультатЗапроса = Запрос.Выполнить();
        // Обработка результата
        Для Каждого СтрокаРезультата Из РезультатЗапроса.Выгрузить() Цикл
            // Обработка данных
        КонецЦикла;
        ```
*   **Форматирование запросов:**
    *   текст запроса форматируется с использованием символа `|` и отступов для читаемости;
    *   обязательно использование параметров (`&Задача`) вместо подстановки значений в текст;
    *   `РАЗРЕШЕННЫЕ РАЗЛИЧНЫЕ` используется там, где это необходимо для оптимизации прав доступа.

### 6.3. Коллекции
*   При работе с табличными частями используются `НайтиСтроки(СтруктураПоиска)`, `СвернутьМассив`, `ОтборСтрок`.

## 7. Интерфейс, асинхронность и локализация

*   **Асинхронные диалоги:** все диалоги, открытие форм и асинхронные операции реализуются через `ОписаниеОповещения`.
    *   *Пример:* `ОписаниеОповещения = Новый ОписаниеОповещения("ПрикреплениеФайла_Завершение", ЭтотОбъект);`
*   **Локализация (НСтр):** все строки, отображаемые пользователю (заголовки диалогов, сообщения, вопросы), оборачиваются в `НСтр("ru = '...'")`.
*   **Управление интерфейсом:** скрытие/показ и доступность элементов формы централизованно управляются в процедуре `УправлениеДоступностью(Форма)`, которая вызывается при изменении ключевых реквизитов и при создании формы.
*   **Предопределённые значения:** для сравнения с предопределёнными элементами справочников, перечислений и видов субконто используется `ПредопределенноеЗначение("...")`.

## 8. Обработка событий форм и табличных частей

*   **Удаление строк:** при `ПередУдалением` проверяется наличие связанных данных в других табличных частях или регистрах. При `ПослеУдаления` обновляются отборы и пересчитываются итоги.

## 9. Архитектурные принципы

*   **Идемпотентность:** методы инициализации и обновления состояния формы (например, `УправлениеДоступностью`) безопасны для многократного вызова.
*   **Инкапсуляция логики:** сложные алгоритмы (например, восстановление потерянных связей) выносятся в отдельные процедуры, а не объединяются внутри обработчиков событий.
*   **Работа с датами:** используется `НачалоДня(ТекущаяДата())` или `ТекущаяДатаСеанса()` для исключения проблем со временем. Пустые даты инициализируются через `Дата(1, 1, 1)` или `ПустаяДата()`.
