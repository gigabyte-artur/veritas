# Передача табличных данных между формами 1С (массив структур / ValueTable)

Целевой проект: `D:\1C\VeritasGit\Veritas\DataProcessors\ТестированиеЗнаний`

## Выбор способа передачи

Существует **два** рабочих способа передать набор записей из одной формы в другую:

| Способ | Тип параметра | Передача с клиента | Когда использовать |
|---|---|---|---|
| **Массив структур** (рекомендуемый) | `v8:Array` | `Новый Массив` → `Новый Структура` | Всегда, когда формируете данные на клиенте |
| **ValueTable** | `v8:ValueTable` | Только на сервере | Когда передача происходит целиком на сервере |

> **⚠️ Важно для мобильного клиента:** `Новый ТаблицаЗначений` недоступен на клиенте. Собирать данные для передачи между формами нужно через **массив структур**. ValueTable можно использовать только при передаче сервер-сервер (например, в серверных вызовах).

**Рекомендация:** на клиенте всегда используйте **массив структур** — это единственный надёжный способ, одинаково работающий на всех типах клиентов (толстый, тонкий, мобильный).

### ⚠️ Важное требование

**Параметры формы ОБЯЗАТЕЛЬНО должны иметь указанный тип** в секции `<Parameters>`:

```xml
<!-- ПРАВИЛЬНО: тип указан -->
<Parameter name="МоиДанные">
    <Type>
        <v8:Type>v8:ValueTable</v8:Type>
    </Type>
</Parameter>

<!-- НЕПРАВИЛЬНО: тип не указан — конфигурация не загрузится -->
<Parameter name="МоиДанные"/>
```

Параметры без `<Type>` приводят к ошибке загрузки конфигурации из XML-файлов.

## Контекст

Форма содержит несколько таблиц-реквизитов типа `v8:ValueTable` с одинаковой структурой колонок. Требуется собрать данные из всех таблиц и передать в другую форму единым массивом структур.

Пример: `СтопкаДобора`, `РукаИгрока`, `СтопкаСброса` — все имеют колонки `Карта` (ссылка) + `ИдентификаторКарты` (строка). На этапе "Торговец" карты из всех трёх стопок передаются в форму торговца как параметр `КартыИгрока`.

## Шаг 1. Определить структуру колонок таблиц-источников

Открыть `Forms\ИмяФормы\Ext\Form.xml` и найти секцию `<Attribute>` для каждой таблицы:

```xml
<Attribute name="СтопкаДобора" id="N">
  <Type><v8:Type>v8:ValueTable</v8:Type></Type>
  <Columns>
    <Column name="Карта" id="1">
      <Type><v8:Type>cfg:CatalogRef.Карты</v8:Type></Type>
    </Column>
    <Column name="ИдентификаторКарты" id="2">
      <Type><v8:Type>xs:string</v8:Type>…</Type>
    </Column>
  </Columns>
</Attribute>
```

Это даёт точные имена и типы колонок для построения структур.

## Шаг 2. Собрать данные из всех таблиц в массив структур

В вызывающей процедуре (клиентской, `&НаКлиенте`):

```bsl
// Получаем все карты игрока из стопок
МассивДанных = Новый Массив;
Для Каждого ТекСтрока Из Таблица1 Цикл
    СтруктураЭлемента = Новый Структура;
    СтруктураЭлемента.Вставить("Поле1", ТекСтрока.Поле1);
    СтруктураЭлемента.Вставить("Поле2", ТекСтрока.Поле2);
    МассивДанных.Добавить(СтруктураЭлемента);
КонецЦикла;
Для Каждого ТекСтрока Из Таблица2 Цикл
    СтруктураЭлемента = Новый Структура;
    СтруктураЭлемента.Вставить("Поле1", ТекСтрока.Поле1);
    СтруктураЭлемента.Вставить("Поле2", ТекСтрока.Поле2);
    МассивДанных.Добавить(СтруктураЭлемента);
КонецЦикла;
// Аналогично для Таблица3, Таблица4 и т.д.
```

**Правила из AI_RULES.md:**
- Каждое действие на отдельной строке (запрет составных выражений)
- Пустые строки внутри процедур/функций не допускаются
- Имена переменных: CamelCase, осмысленные

## Шаг 3. Добавить параметр в структуру и открыть форму

```bsl
ПараметрыФормы = Новый Структура;
ПараметрыФормы.Вставить("Параметр1", Значение1);
ПараметрыФормы.Вставить("Параметр2", Значение2);
ПараметрыФормы.Вставить("ИмяМассива", МассивДанных);  // ключевой параметр
ОписаниеОЗакрытии = Новый ОписаниеОповещения("Обработчик_ПослеЗакрытия", ЭтотОбъект);
ОткрытьФорму("Обработка.ИмяОбработки.Форма.ИмяФормы",
    ПараметрыФормы, ЭтаФорма, , , , ОписаниеОЗакрытии, РежимОткрытияОкнаФормы.БлокироватьОкноВладельца);
```

**Важно:** `РежимОткрытияОкнаФормы.БлокироватьОкноВладельца` — модальный режим.

## Шаг 4. Определить параметр в целевой форме (Form.xml)

Открыть `Forms\ЦелеваяФорма\Ext\Form.xml` и добавить в секцию `<Parameters>`:

```xml
<Parameter name="ИмяМассива">
    <Type>
        <v8:Type>v8:Array</v8:Type>
    </Type>
</Parameter>
```

**Ключевые моменты:**
- Тип параметра — `v8:Array` (не `v8:ValueListType`, не `v8:ValueTable`)
- Имя должно точно совпадать с ключом в `Структура.Вставить("ИмяМассива", ...)` (регистр важен)

## Шаг 5. Прочитать параметр в модуле целевой формы

В процедуре `ПриСозданииНаСервере` целевой формы:

```bsl
&НаСервере
Процедура ПриСозданииНаСервере(Отказ, СтандартнаяОбработка)
    // Чтение обычных параметров
    Параметр1 = Параметры.Параметр1;
    // Чтение массива структур
    МассивНаФорме = Параметры.ИмяМассива;
    // ... остальная инициализация
КонецПроцедуры
```

Для этого реквизит `МассивНаФорме` должен быть объявлен в секции `<Attributes>` целевой формы как тип `v8:Array` (или другой подходящий тип). Если массив загружается только для передачи в дочерние методы, реквизит не обязателен — достаточно переменной модуля.

## Ограничения

- Массив структур не типизирован на уровне XML (все структуры могут иметь разный набор ключей)
- Для безопасного чтения ключа структуры используйте `ОбщегоНазначенияКлиентСервер.СвойствоСтруктуры(Структура, "Ключ", ЗначениеПоУмолчанию)`
- Если в целевой форме требуется изменять массив и возвращать обратно — используйте реквизит формы (тип `v8:Array`) с синхронизацией через сервер
- Этот паттерн не подходит для очень больших объёмов данных (сотни тысяч строк) — используйте запросы и временные таблицы

## Альтернатива: передача ValueTable напрямую

> **⚠️ Внимание:** `ТаблицаЗначений` недоступна на клиенте. Этот способ работает **только** при передаче данных с сервера на сервер (например, внутри `&НаСервере` процедуры). Если данные собираются на клиенте — используйте массив структур (см. выше).

Вместо массива структур можно создать `ТаблицаЗначений` на сервере и передать её как параметр с типом `v8:ValueTable`. Это удобно, когда принимающая форма имеет реквизиты-ValueTable с теми же колонками.

### Вызывающая сторона (клиент)

```bsl
&НаКлиенте
Процедура ОткрытьФормуСТаблицей()
    // Инициализация.
    ПараметрыФормы = Новый Структура;
    // Создание таблицы с нужными колонками
    ТаблицаДанных = Новый ТаблицаЗначений;
    ТаблицаДанных.Колонки.Добавить("Ссылка");
    ТаблицаДанных.Колонки.Добавить("Значение");
    // Заполнение из реквизита-ValueTable текущей формы
    Для Каждого ТекСтрока Из ИсходнаяТаблица Цикл
        НоваяСтрока = ТаблицаДанных.Добавить();
        НоваяСтрока.Ссылка = ТекСтрока.Ссылка;
        НоваяСтрока.Значение = ТекСтрока.Значение;
    КонецЦикла;
    ПараметрыФормы.Вставить("ТаблицаПараметр", ТаблицаДанных);
    // Открытие формы с блокировкой окна владельца
    ОткрытьФорму("Обработка.ТестированиеЗнаний.Форма.ЦелеваяФорма",
        ПараметрыФормы, ЭтаФорма, , , , , РежимОткрытияОкнаФормы.БлокироватьОкноВладельца);
КонецПроцедуры
```

### Принимающая сторона (параметр в Form.xml)

```xml
<Parameters>
    <Parameter name="ТаблицаПараметр">
        <Type>
            <v8:Type>v8:ValueTable</v8:Type>
        </Type>
    </Parameter>
</Parameters>
```

### Принимающая сторона (модуль, ПриСозданииНаСервере)

```bsl
&НаСервере
Процедура ПриСозданииНаСервере(Отказ, СтандартнаяОбработка)
    // Прямое присваивание: таблица из параметра становится реквизитом формы
    РеквизитТаблица = Параметры.ТаблицаПараметр;
КонецПроцедуры
```

**Важно:** имена колонок в `ТаблицаЗначений` (созданной на клиенте) должны **точно совпадать** с именами колонок реквизита-ValueTable в целевой форме, иначе произойдёт ошибка присваивания. Колонки на клиенте создаются без указания типа (простым `Добавить("Имя")`) — платформа сама приводит типы при передаче на сервер.
