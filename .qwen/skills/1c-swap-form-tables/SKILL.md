# Обмен содержимого двух табличных реквизитов формы (1С, управляемые формы)

Целевой проект: `D:\1C\VeritasGit\Veritas\DataProcessors\ТестированиеЗнаний\Forms\Форма\Ext\Form\Module.bsl`

## Контекст

В форме есть два табличных реквизита (ValueTable) с идентичной структурой колонок. Требуется поменять их содержимое местами — все строки из таблицы А перенести в таблицу Б, а из Б в А.

**Особенность:** операция выполняется на клиенте (`&НаКлиенте`), без серверного контекста. Нельзя использовать `Выгрузить()` / `Загрузить()` на прямую, т.к. при последующем вызове серверного метода изменения могут не синхронизироваться корректно.

## Паттерн: массивы структур для сохранения строк

### Шаг 1. Сохранить содержимое обеих таблиц в массивы структур

Для каждой строки таблицы создать структуру с теми же полями, что и колонки таблицы:

```bsl
// Сохраняем карты из обеих стопок в массивы структур
МассивТаблицаА = Новый Массив;
Для Каждого ТекСтрока Из ТаблицаА Цикл
	СтруктураСтроки = Новый Структура;
	СтруктураСтроки.Вставить("Поле1", ТекСтрока.Поле1);
	СтруктураСтроки.Вставить("Поле2", ТекСтрока.Поле2);
	// ... остальные поля
	МассивТаблицаА.Добавить(СтруктураСтроки);
КонецЦикла;

МассивТаблицаБ = Новый Массив;
Для Каждого ТекСтрока Из ТаблицаБ Цикл
	СтруктураСтроки = Новый Структура;
	СтруктураСтроки.Вставить("Поле1", ТекСтрока.Поле1);
	СтруктураСтроки.Вставить("Поле2", ТекСтрока.Поле2);
	// ... остальные поля
	МассивТаблицаБ.Добавить(СтруктураСтроки);
КонецЦикла;
```

**Правила:**
- Каждое действие на отдельной строке (запрет составных выражений — ред.) — да, `Структура.Вставить()` это один вызов
- Имена полей в структуре должны точно совпадать с именами колонок таблицы (регистр)
- Для таблиц с большим числом колонок перечислять все поля явно

### Шаг 2. Очистить обе таблицы

```bsl
ТаблицаА.Очистить();
ТаблицаБ.Очистить();
```

### Шаг 3. Заполнить таблицы в обратном порядке

```bsl
// Заполняем таблицу А строками из бывшей таблицы Б
Для Каждого ТекСтрока Из МассивТаблицаБ Цикл
	НоваяСтрока = ТаблицаА.Добавить();
	НоваяСтрока.Поле1 = ТекСтрока.Поле1;
	НоваяСтрока.Поле2 = ТекСтрока.Поле2;
	// ... остальные поля
КонецЦикла;

// Заполняем таблицу Б строками из бывшей таблицы А
Для Каждого ТекСтрока Из МассивТаблицаА Цикл
	НоваяСтрока = ТаблицаБ.Добавить();
	НоваяСтрока.Поле1 = ТекСтрока.Поле1;
	НоваяСтрока.Поле2 = ТекСтрока.Поле2;
	// ... остальные поля
КонецЦикла;
```

### Шаг 4. Продолжить клиентскую логику

После замены таблиц можно вызывать серверные методы формы — изменения будут синхронизированы при ближайшем серверном вызове:

```bsl
// Игрок берёт одну карту из стопки добора (серверный метод)
ДобратьКартуВРуку();
```

## Полный пример из проекта Veritas (карта «Фокус»)

Механика: меняет местами стопки сброса и добора, после чего игрок берёт 1 карту.

```bsl
// Выполняет действие карты Фокус.
// Меняет местами стопки сброса и добора, после чего игрок берёт одну карту из стопки добора.
//
// Параметры:
//  НайденнаяКарта - СправочникСсылка.Карты - Карта, которую разыгрываем.
//  ИдентификаторКартыВход - Строка - Идентификатор карты, которую разыгрываем.
//
&НаКлиенте
Процедура ДействиеКарты_Фокус(НайденнаяКарта, ИдентификаторКартыВход)
	// Сохраняем карты из обеих стопок в массивы структур
	МассивСтопкаСброса = Новый Массив;
	Для Каждого ТекСтрока Из СтопкаСброса Цикл
		СтруктураКарты = Новый Структура;
		СтруктураКарты.Вставить("Карта", ТекСтрока.Карта);
		СтруктураКарты.Вставить("ИдентификаторКарты", ТекСтрока.ИдентификаторКарты);
		МассивСтопкаСброса.Добавить(СтруктураКарты);
	КонецЦикла;
	МассивСтопкаДобора = Новый Массив;
	Для Каждого ТекСтрока Из СтопкаДобора Цикл
		СтруктураКарты = Новый Структура;
		СтруктураКарты.Вставить("Карта", ТекСтрока.Карта);
		СтруктураКарты.Вставить("ИдентификаторКарты", ТекСтрока.ИдентификаторКарты);
		МассивСтопкаДобора.Добавить(СтруктураКарты);
	КонецЦикла;
	// Очищаем обе стопки
	СтопкаСброса.Очистить();
	СтопкаДобора.Очистить();
	// Заполняем стопку добора картами из бывшей стопки сброса
	Для Каждого ТекКарта Из МассивСтопкаСброса Цикл
		НоваяСтрока = СтопкаДобора.Добавить();
		НоваяСтрока.Карта = ТекКарта.Карта;
		НоваяСтрока.ИдентификаторКарты = ТекКарта.ИдентификаторКарты;
	КонецЦикла;
	// Заполняем стопку сброса картами из бывшей стопки добора
	Для Каждого ТекКарта Из МассивСтопкаДобора Цикл
		НоваяСтрока = СтопкаСброса.Добавить();
		НоваяСтрока.Карта = ТекКарта.Карта;
		НоваяСтрока.ИдентификаторКарты = ТекКарта.ИдентификаторКарты;
	КонецЦикла;
	// Игрок берёт одну карту из стопки добора
	ДобратьКартуВРуку();
КонецПроцедуры
```

## Когда использовать

- Когда нужно полностью обменять содержимое двух табличных реквизитов формы
- Когда операция должна выполняться на клиенте (директива `&НаКлиенте`)
- Когда размер таблиц невелик (десятки-сотни строк) — для больших объёмов данных лучше серверный подход с `Выгрузить()`/`Загрузить()`

## Альтернатива (чисто серверный подход)

Если процедура серверная (`&НаСервере`), можно проще:

```bsl
&НаСервере
Процедура ПоменятьСтопкиМестами()
	Сохранение = СтопкаА.Выгрузить();
	СтопкаА.Загрузить(СтопкаБ.Выгрузить());
	СтопкаБ.Загрузить(Сохранение);
КонецПроцедуры
```

Однако клиентский подход с массивами структур предпочтительнее, когда:
1. После замены нужно продолжить клиентскую логику без лишнего серверного вызова
2. Серверный контекст не требуется для остальной части процедуры
3. `Выгрузить()`/`Загрузить()` проблематичны из-за синхронизации контекста
