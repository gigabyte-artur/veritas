# Рефакторинг управляемой формы 1С: Pages → единый экран (для игр/боёв)

Контекст: проект Veritas (1С:Предприятие 8.3.26+, управляемые формы, мобильный клиент).  
Исходная форма использует `<Pages>` с двумя вкладками: "Вопрос" и "Об игре" — игрок не видит контекст боя (HP, энергия, намерение врага) одновременно с вопросом.

## Целевая архитектура

Форма содержит одну вертикальную группу с четырьмя зонами (без Pages):

```
┌──────────────────────────────┐
│  Зона 1: HUD_Верх            │  ~15% высоты, Horizontal
│  [Player: ❤ HP/MAX ██░░░]   │
│  [Enemy: name intent ⚔]     │
│  [Streak: ★ ★ ★]            │
├──────────────────────────────┤
│  Зона 2: Арена_Вопроса       │  растягивается (stretch)
│  [картинка задания]          │
│  [текст вопроса]             │
├──────────────────────────────┤
│  Зона 3: Ответы_Сетка        │  2 колонки кнопок
│  [1] [2] [3] │ [4] [5] [6]  │
├──────────────────────────────┤
│  Зона 4: Рука_Карт           │  динамические кнопки карт
│  [⚡Карта] [⚡Карта] [⚡Карта] │
└──────────────────────────────┘
```

## Шаг 1. Замена Pages в Form.xml

### 1.1 Удалить
- `<Pages name="ГруппаСтраницы">` целиком (обе страницы и всё их содержимое)
- Табличные элементы `ПараметрыИгрока`, `ПараметрыПротивника`, `Модификаторы` (как визуальные таблицы; реквизиты остаются)

### 1.2 Добавить 4 зоны

**Зона 1 — HUD_Верх:**

```xml
<UsualGroup name="HUD_Верх" id="200">
    <VerticalStretch>false</VerticalStretch>
    <Group>Horizontal</Group>
    <Representation>None</Representation>
    <ShowTitle>false</ShowTitle>
    <ChildItems>
        <!-- Панель игрока: сердце + HP + прогресс-бар + щит + энергия -->
        <!-- Панель противника: имя + намерение + HP + прогресс-бар -->
        <!-- Индикатор серии: 3 PictureDecoration (заряд/пусто) -->
    </ChildItems>
</UsualGroup>
```

Ключевые элементы HUD:
- **Прогресс-бар здоровья:** `LabelDecoration` с текстом из 20 символов `█`/`░`, обновляется программно
- **Щит/энергия:** `LabelDecoration` с пустым заголовком, обновляется через `.Заголовок`
- **Индикатор серии:** 3 `PictureDecoration` с `CommonPicture.Звезда` (пусто) и `CommonPicture.Серия3` (заряд)

**Зона 2 — Арена_Вопроса:**
```xml
<UsualGroup name="Арена_Вопроса" id="280">
    <HorizontalStretch>true</HorizontalStretch>
    <VerticalStretch>true</VerticalStretch>
    <Group>Vertical</Group>
    <ChildItems>
        <PictureField name="АдресКартинки" id="28">...</PictureField>
        <InputField name="ЗаданиеТекстВопросаЗадания" id="25">...</InputField>
    </ChildItems>
</UsualGroup>
```

**Зона 3 — Ответы_Сетка:**
```xml
<UsualGroup name="Ответы_Сетка" id="290">
    <HorizontalStretch>true</HorizontalStretch>
    <Group>Horizontal</Group>
    <ChildItems>
        <UsualGroup name="Ответы_КолонкаЛевая" id="292">...кнопки 1,2,3...</UsualGroup>
        <UsualGroup name="Ответы_КолонкаПравая" id="294">...кнопки 4,5,6...</UsualGroup>
    </ChildItems>
</UsualGroup>
```

**Зона 4 — Рука_Карт (пустая, заполняется динамически):**
```xml
<UsualGroup name="Рука_Карт" id="296">
    <VerticalStretch>false</VerticalStretch>
    <Group>Horizontal</Group>
</UsualGroup>
```

## Шаг 2. Новые процедуры в Module.bsl

### 2.1 ОбновитьИнтерфейсБоя()
Вызывается при каждом изменении состояния (урон, лечение, смена хода).

```bsl
&НаКлиенте
Процедура ОбновитьИнтерфейсБоя()
    // Рассчитать прогресс-бары (20 блоков █/░)
    ПрогрессЗдоровьяИгрокаПроценты = (ТекущееЗдоровьеИгрока / МаксимальноеЗдоровьеИгрока) * 100;
    Заполнение = Цел(ПрогрессЗдоровьяИгрокаПроценты / 100 * 20);
    // Построить строку из 20 символов
    ТекстБара = "";
    Для Сч = 1 По 20 Цикл
        ТекстБара = ТекстБара + ?(Сч <= Заполнение, "█", "░");
    КонецЦикла;
    Элементы.HUD_БарЗдоровьяИгрока.Заголовок = ТекстБара;
    // Цвет: красный при HP < 30%
    Элементы.HUD_БарЗдоровьяИгрока.ЦветТекстаЗаголовка = ?(ТекущееЗдоровьеИгрока < МаксимальноеЗдоровьеИгрока * 0.3, WebЦвета.Красный, WebЦвета.ЗеленыйЛес);
    // Аналогично для противника
    // Обновить текст щита и энергии
    Элементы.HUD_ТекстЩита.Заголовок = Строка(ПолучитьЗначениеПараметраИгрока(ВидПараметраЩит));
    Элементы.HUD_ТекстЭнергии.Заголовок = Строка(ПолучитьЗначениеПараметраИгрока(ВидПараметраЭнергия));
    ОбновитьИндикаторСерии();
КонецПроцедуры
```

### 2.2 ОбновитьИндикаторСерии()
Переключает 3 иконки между `CommonPicture.Звезда` и `CommonPicture.Серия3`.

```bsl
&НаКлиенте
Процедура ОбновитьИндикаторСерии()
    Элементы.HUD_ЯчейкаСерии1.Картинка = ?(ЗарядСерии >= 1, CommonPicture.Серия3, CommonPicture.Звезда);
    Элементы.HUD_ЯчейкаСерии2.Картинка = ?(ЗарядСерии >= 2, CommonPicture.Серия3, CommonPicture.Звезда);
    Элементы.HUD_ЯчейкаСерии3.Картинка = ?(ЗарядСерии >= 3, CommonPicture.Серия3, CommonPicture.Звезда);
КонецПроцедуры
```

### 2.3 ОбновитьРукуКарт() — переиспользование предсозданных слотов

**⚠️ Важно:** `Элементы.Добавить()` в рантайме НЕ работает для кнопок (ошибка "Изменение состава элементов недоступно"). Вместо этого:
1. В `Form.xml` заранее создаются N кнопок-слотов (например, `КартаСлот1..10`) внутри группы `Рука_Карт`
2. В коде слоты показываются/скрываются и перезаполняются

**Form.xml для зоны Рука_Карт (N=10):**
```xml
<UsualGroup name="Рука_Карт" id="296">
    <VerticalStretch>false</VerticalStretch>
    <Group>Horizontal</Group>
    <Representation>None</Representation>
    <ShowTitle>false</ShowTitle>
    <ChildItems>
        <Button name="КартаСлот1" id="300">
            <Type>UsualButton</Type>
            <Height>3</Height><Width>8</Width>
            <CommandName>Form.Command.КартаСлот1</CommandName>
            <Title><v8:item><v8:lang>ru</v8:lang><v8:content>-</v8:content></v8:item></Title>
        </Button>
        <!-- КартаСлот2..10 аналогично -->
    </ChildItems>
</UsualGroup>
```

**Команды в Form.xml — все слоты ведут на один обработчик:**
```xml
<Command name="КартаСлот1" id="13">
    <Action>КартаСлот_Нажатие</Action>
    <CurrentRowUse>DontUse</CurrentRowUse>
</Command>
<!-- КартаСлот2..10 аналогично -->
```

**Module.bsl — ОбновитьРукуКарт():**
```bsl
&НаКлиенте
Процедура ОбновитьРукуКарт()
    МассивСлотов = Новый Массив;
    МассивСлотов.Добавить("КартаСлот1");
    // ... добавить все имена слотов
    // Скрыть все слоты
    Для Каждого ИмяСлота Из МассивСлотов Цикл
        Элементы[ИмяСлота].Видимость = Ложь;
    КонецЦикла;
    // Заполнить по количеству карт в руке
    Индекс = 0;
    Для Каждого КартаИзРуки Из РукаИгрока Цикл
        Если Индекс >= МассивСлотов.Количество() Тогда Прервать; КонецЕсли;
        Элемент = Элементы[МассивСлотов[Индекс]];
        Элемент.Видимость = Истина;
        Элемент.Заголовок = КартаИзРуки.Карта.Наименование + "...⚡" + Строка(КартаИзРуки.Карта.Стоимость);
        Элемент.Доступность = (ЭнергияИгрока >= КартаИзРуки.Карта.Стоимость);
        // Окрасить рамку по типу карты
        Элемент.УстановитьОформление(Новый Оформление(, ЦветРамки));
        Индекс = Индекс + 1;
    КонецЦикла;
КонецПроцедуры
```

**Module.bsl — единый обработчик нажатия:**
```bsl
&НаКлиенте
Процедура КартаСлот_Нажатие(Элемент, СтандартнаяОбработка)
    СтандартнаяОбработка = Ложь;
    ИмяЭлемента = Элемент.Имя;
    НомерСлота = Число(Прав(ИмяЭлемента, СтрДлина(ИмяЭлемента) - 9)); // "КартаСлот"=9 символов
    ИндексКарты = НомерСлота - 1;
    Если ИндексКарты >= 0 И ИндексКарты < РукаИгрока.Количество() Тогда
        РазыгратьКарту(РукаИгрока[ИндексКарты].ИдентификаторКарты);
        ОбновитьРукуКарт();
        ОбновитьИнтерфейсБоя();
    КонецЕсли;
КонецПроцедуры
```

## Шаг 3. Патчи существующих процедур

| Процедура | Добавить вызов |
|---|---|
| `ОтобразитьСледующееЗадание()` | `ОбновитьИнтерфейсБоя()` + `ОбновитьРукуКарт()` |
| `УвеличитьЗарядСерии()` | `ОбновитьИндикаторСерии()` + `ОбновитьИнтерфейсБоя()` |
| `ОтобразитьСостояниеИгрока()` | `ОбновитьИнтерфейсБоя()` |
| `ОтобразитьСостояниеПротивника()` | `ОбновитьИнтерфейсБоя()` |
| `ОбработатьВыбранныйВидЭтапа()` (после добора руки) | `ОбновитьРукуКарт()` |
| `УправлениеВидимостью()` | Заменить `Элементы.ГруппаСтраницаОбИгре` на `Элементы.HUD_Верх`, `Элементы.Рука_Карт` |

## Шаг 4. Прогресс-бары через Unicode

Используйте 20 символов для отображения прогресса:

```bsl
Заполнение = Цел(Процент / 100 * 20);
ТекстБара = "";
Для Сч = 1 По 20 Цикл
    ТекстБара = ТекстБара + ?(Сч <= Заполнение, "█", "░");
КонецЦикла;
Элементы.ИмяБара.Заголовок = ТекстБара;
Элементы.ИмяБара.ЦветТекстаЗаголовка = ЦветБара;
```

## Шаг 5. Проверка перед сдачей

- [ ] `УправлениеВидимостью()` не ссылается на удалённые элементы (ГруппаСтраницы, ГруппаСтраницаОбИгре)
- [ ] Все новые элементы HUD имеют уникальные ID (не пересекаются с существующими)
- [ ] Старые элементы (АдресКартинки, ЗаданиеТекстВопросаЗадания, кнопки ответов) перенесены в новые зоны с сохранением ID
- [ ] Бизнес-логика (расчёт урона, действия карт, цикл противника) не изменена
- [ ] Для не-игровых режимов HUD и рука карт скрыты через `УправлениеВидимостью()`
