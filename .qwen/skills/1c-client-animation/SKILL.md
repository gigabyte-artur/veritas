# Плавная анимация значений на клиенте 1С

Используется для визуального эффекта плавного изменения числового реквизита формы (например, здоровья, энергии, щита) на клиенте, когда серверное значение уже зафиксировано мгновенно.

## Ключевые ограничения платформы

1. **`ПодключитьОбработчикОжидания` с задержкой < 1 требует однократного исполнения**. Если второй параметр < 1 (например, 0.1 для 100 мс), третий параметр `ОднократноеИсполнение` принудительно `Истина`. Это значит, что обработчик вызывается один раз, и для повторения нужно переподключать его вручную на каждом шаге.

2. **После серверного вызова платформа может синхронизировать реквизиты формы**. Даже если серверная функция не присваивает реквизит явно, при возврате на клиент его значение может быть перечитано из контекста, что вызывает "артефакт" — мгновенный скачок значения до целевого перед запуском анимации.

## Решение проблемы синхронизации (флаг блокировки)

Добавьте на форму реквизит `ЗапретИзмененияРеквизита` (Булево). В серверной процедуре, которая синхронизирует реквизит (например, `ОтобразитьСостояниеПротивника()`), проверяйте этот флаг:

```bsl
&НаСервере
Процедура ОтобразитьСостояние()
    Если НЕ ЗапретИзмененияРеквизита Тогда
        РеквизитФормы = ПолучитьЗначениеССервера();
    КонецЕсли;
    // остальные реквизиты обновляются как обычно
    ОбновитьИнтерфейс();
КонецПроцедуры
```

На клиенте перед анимацией установите флаг в `Истина`, после завершения — в `Ложь`.

## Переменные модуля

```bsl
&НаКлиенте
Перем АнимСтарт;   // Стартовое значение до анимации
&НаКлиенте
Перем АнимЦель;    // Целевое значение (финальное)
&НаКлиенте
Перем АнимТек;     // Текущее отображаемое значение на каждом шаге
```

## Шаг анимации (однократный, переподключается вручную)

Поскольку задержка < 1, обработчик одноразовый. Для продолжения анимации — переподключаем его на каждом шаге.

```bsl
&НаКлиенте
Процедура ШагАнимации()
    АнимТек = АнимТек - 1;                     // или +1 для увеличения
    РеквизитФормы = АнимТек;                    // устанавливаем реквизит
    ОбновитьЭлементНаКлиенте();                  // перерисовываем UI
    Если АнимТек <= АнимЦель Тогда               // условие завершения
        РеквизитФормы = АнимЦель;                // финальная синхронизация
        ОбновитьЭлементНаКлиенте();
        ЗапретИзмененияРеквизита = Ложь;         // сброс флага
    Иначе
        // Переподключаем на следующий шаг
        ПодключитьОбработчикОжидания("ШагАнимации", 0.1, Истина);
    КонецЕсли;
КонецПроцедуры
```

## Функция-обёртка

**Важно:** не делайте серверный вызов для получения целевого значения после серверной функции. Вычислите его из результата серверной функции: `АнимЦель = АнимСтарт - Результат.УронПоЗдоровью`. Это предотвращает лишнюю синхронизацию реквизита.

```bsl
&НаКлиенте
Функция ПлавноеИзменение(ЗначениеИзменения)
    // Подготовка: флаг блокирует синхронизацию при серверных вызовах
    ЗапретИзмененияРеквизита = Истина;

    // Защита: отключить предыдущую анимацию, если запущена новая
    Попытка
        ОтключитьОбработчикОжидания("ШагАнимации");
    Исключение
        // Обработчик не был подключён — нормально
    КонецПопытки;

    // Запомнить стартовое отображаемое значение
    АнимСтарт = РеквизитФормы;

    // Вызвать серверную функцию (данные фиксируются мгновенно)
    // Флаг = Истина, поэтому реквизит не синхронизируется на клиенте
    Результат = СервернаяФункция(ЗначениеИзменения);

    // Восстановить реквизит на случай синхронизации платформой
    РеквизитФормы = АнимСтарт;

    // Вычислить финальное значение из результата серверной функции
    // (избегаем лишнего серверного вызова!)
    АнимЦель = АнимСтарт - Результат.УронПоЗдоровью;
    Если АнимЦель < 0 Тогда АнимЦель = 0; КонецЕсли;
    АнимТек = АнимСтарт;

    // Если значения совпадают — анимация не нужна
    Если АнимСтарт <= АнимЦель Тогда
        РеквизитФормы = АнимЦель;
        ОбновитьЭлементНаКлиенте();
        ЗапретИзмененияРеквизита = Ложь;
        Возврат Результат;
    КонецЕсли;

    // Запустить анимацию: каждые 100 мс (0.1), одноразовый режим
    ПодключитьОбработчикОжидания("ШагАнимации", 0.1, Истина);

    Возврат Результат;
КонецФункции
```

## Полный алгоритм

1. Установить флаг `ЗапретИзмененияРеквизита = Истина` (предотвращает синхронизацию из любого серверного вызова)
2. Запомнить `АнимСтарт = РеквизитФормы` (текущее отображаемое значение)
3. Вызвать серверную функцию, которая мгновенно фиксирует новое значение данных
4. Восстановить `РеквизитФормы = АнимСтарт` (защита от синхронизации платформой)
5. Вычислить `АнимЦель` из результата серверной функции (без лишнего серверного вызова)
6. Если `АнимСтарт <= АнимЦель` — анимация не нужна, сразу установить значение, сбросить флаг
7. Иначе — запустить одноразовый `ПодключитьОбработчикОжидания`
8. В обработчике: уменьшить/увеличить → обновить UI → если не достигли цели, переподключить обработчик; если достигли — сбросить флаг блокировки

## Ключевые особенности

1. **Серверные вызовы не трогать** — `ПодключитьОбработчикОжидания` работает только на клиенте. Серверные процедуры (действия карт, расчёты) продолжают работать мгновенно.
2. **Защита от повторных ударов** — при новом вызове во время анимации старый обработчик отключается (`Попытка...Исключение`), и анимация запускается заново от текущего отображаемого положения.
3. **UI обновление изолировано** — клиентская процедура обновления элемента (прогресс-бар, текст) отделена от серверной `ОбновитьИнтерфейс()`, что позволяет перерисовывать его без серверного вызова.
4. **Флаг блокировки** — реквизит формы `ЗапретИзмененияРеквизита` защищает анимируемый реквизит от синхронизации при любых серверных вызовах во время анимации. Должен быть сброшен в `Ложь` после завершения анимации.
5. **Вычисление цели из результата** — вместо повторного серверного вызова для получения финального значения, вычисляйте его из результата первой серверной функции. Это устраняет лишний серверный вызов, который мог бы синхронизировать реквизит.
