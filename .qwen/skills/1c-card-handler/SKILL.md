# Добавление новой карты в обработку ТестированиеЗнаний (1С)

Целевой файл: `DataProcessors\ТестированиеЗнаний\Forms\Форма\Ext\Form\Module.bsl`

## Шаг 1. Найти точку входа

Процедура-диспетчер: `ДействиеВыбраннойКарты(НайденнаяКарта, ИдентификаторКартыВход)`

Диспетчер построен как цепочка `Если / ИначеЕсли` с проверкой через
`ПредопределенноеЗначение("Справочник.Карты.ИмяКарты")`:

```bsl
ИначеЕсли НайденнаяКарта = ПредопределенноеЗначение("Справочник.Карты.ИмяКарты") Тогда
    ДействиеКарты_ИмяКарты(НайденнаяКарта, ИдентификаторКартыВход);
```

Новую ветку добавлять **перед блоком `Иначе`** (последняя `ИначеЕсли`).

## Шаг 2. Создать процедуру действия

Поместить между существующими процедурами (например, после `ДействиеКарты_ЧистыйКонтракт`).

Сигнатура:
```bsl
&НаСервере
Процедура ДействиеКарты_ИмяКарты(НайденнаяКарта, ИдентификаторКартыВход)
    // логика
КонецПроцедуры
```

Большинство обработчиков карт выполняются на сервере (аннотация `&НаСервере`), но некоторые — на клиенте (`&НаКлиенте`), если им нужно вызывать серверные методы через автоматический серверный вызов платформы:

```bsl
&НаКлиенте
Процедура ДействиеКарты_ИмяКарты(НайденнаяКарта, ИдентификаторКартыВход)
    // Серверный метод можно вызывать напрямую — 1С сама выполнит серверный вызов
    УвеличитьЩитИгрока(7);
    // или
    УвеличитьПараметрИгрока(ВидПараметра, Значение);
КонецПроцедуры
```

**Когда использовать `&НаСервере`:**
- Логика требует прямого доступа к таблицам формы (например, `СтопкаДобора`, `РукаИгрока`)
- Используются серверные генераторы (`Гсч`)
- Сложные расчёты с запросами к БД

**Когда использовать `&НаКлиенте`:**
- Простое изменение параметров игрока/противника (щит, энергия, яд)
- Вызов серверных методов через платформенный автоматический вызов
- Взаимодействие с UI (сообщения, оповещения)

В обоих случаях подпись процедуры одинаковая (принимает `НайденнаяКарта` и `ИдентификаторКартыВход`).

Все функции объявлены в том же модуле формы:

| **Добрать N карт из стопки добора в руку** | **`ДобратьВРукуКарты(Количество)`** — вызывает ДобратьКартуВРуку() в цикле |

### Важное различие: перезапись vs накопление параметров

Параметры (Яд, Горение, Щит, Бомба и др.) могут устанавливаться двумя способами:

| Семантика | Метод | Когда использовать |
|---|---|---|
| **Перезапись** | `УстановитьПараметрПротивника(Вид, N)` | Карта задаёт точное значение (например, "накладывает щит 5") |
| **Накопление** | `УвеличитьПараметрПротивника(Вид, N)` | Карта **добавляет** к текущему значению, эффект стакается |

#### Пример: перезапись (установка точного значения)
```bsl
ВидПараметраЩит = Перечисления.ВидыПараметровИгроков.Щит;
УстановитьПараметрПротивника(ВидПараметраЩит, 5);
```

#### Пример: накопление (увеличение текущего значения)
```bsl
ВидПараметраЯд = Перечисления.ВидыПараметровИгроков.Яд;
УвеличитьПараметрПротивника(ВидПараметраЯд, 6);
```

> **Контекст проекта**: яд у противника накапливается — карты добавляют к текущему значению (через `УвеличитьПараметрПротивника`), а не перезаписывают его.

### Пример: карта, накладывающая эффект на игрока и противника одновременно

```bsl
&НаСервере
Процедура ДействиеКарты_Скорпион(НайденнаяКарта, ИдентификаторКартыВход)
    ВидПараметраЯд = Перечисления.ВидыПараметровИгроков.Яд;
    УвеличитьПараметрПротивника(ВидПараметраЯд, 5);  // +5 яда противнику
    УвеличитьПараметрИгрока(ВидПараметраЯд, 3);      // +3 яда игроку
КонецПроцедуры
```

### Пример: карта, которая даёт взять карту(ы)

Игрок берёт N карт из стопки добора в руку (рекомендуемый способ):

```bsl
// Добрать 1 карту (с автоперемешиванием сброса если стопка пуста)
ДобратьКартуВРуку();

// Добрать 3 карты
ДобратьВРукуКарты(3);
```

Оба метода — серверные (`&НаСервере`), работают с контекстом формы (`СтопкаДобора`, `РукаИгрока`).

> Когда НЕЛЬЗЯ использовать: если после добора нужна ссылка на взятую карту (например, для проверки `ПерваяКарта.Карта.Стоимость`) — используйте прямой `ПереместиКартуИзСтопки()` как раньше.

### Паттерн: взять карты → сбросить случайную (draw + discard)

Пример карт: `Замена` (2 → 1), `СвежийВзгляд` (3 → 1). Сначала добор, потом сброс случайной карты из руки:

```bsl
// Игрок берет N карт из стопки добора в руку
ДобратьВРукуКарты(3);

// Сбрасывает 1 случайную карту из руки в стопку сброса
Если РукаИгрока.Количество() > 0 Тогда
    Индекс = ГсчКлиент.СлучайноеЧисло(0, РукаИгрока.Количество() - 1);
    ВыбраннаяКартаСтрока = РукаИгрока[Индекс];
    ИдентификаторКарты = ВыбраннаяКартаСтрока.ИдентификаторКарты;
    ПереместиКартуИзСтопки(ИдентификаторКарты, "РукаИгрока", "СтопкаСброса", ЭтаФорма);
КонецЕсли;
```

> **Важно:** `ГсчКлиент` используется только на клиенте. Если процедура серверная — используйте `Гсч` (серверный генератор).

Старый (прямой) способ — только когда нужна ссылка на карту:

```bsl
Если СтопкаДобора.Количество() > 0 Тогда
    ПерваяКарта = СтопкаДобора[0];
    ИдентификаторКарты = ПерваяКарта.ИдентификаторКарты;
    ПереместиКартуИзСтопки(ИдентификаторКарты, "СтопкаДобора", "РукаИгрока", ЭтаФорма);
    // Дальше можно использовать ПерваяКарта
КонецЕсли;
```

## Шаг 4. Реализация урона от эффектов (Яд, Горение, Бомба)

При уменьшении параметров противника (в `УменьшитьПараметрыПротивников()`):
1. Урон от яда наносится **до уменьшения** значения параметра
2. Величина урона равна текущему значению яда

Паттерн по аналогии с `УменьшитьПараметрыИгроков()`:

```bsl
ЗначениеПараметра = ПолучитьЗначениеПараметраПротивника(ВидПараметра);
Если ЗначениеПараметра > 0 Тогда
    // Проверка параметра "Яд": наносим урон до уменьшения значения
    Если ВидПараметра = Перечисления.ВидыПараметровИгроков.Яд Тогда
        НанестиУронПротивнику(ЗначениеПараметра);
    КонецЕсли;
    НовоеЗначение = ЗначениеПараметра - 1;
    // ... установка/удаление параметра
КонецЕсли;
```

### Модификатор-зависимые корректировки в УменьшитьПараметры

При изменении шага уменьшения параметра или величины урона от эффекта под влиянием модификатора — проверять активность через `МодификаторАктивен()`.

**Где искать предопределённые модификаторы:**
- `Catalogs\Модификаторы\Ext\Predefined.xml` — список всех предопределённых элементов
- `CommonTemplates\Модификаторы_Предопределенный\Ext\Template.xml` — описания

**Два паттерна обращения к модификатору:**
| Имя в XML (`<Name>`) | Паттерн в коде |
|---|---|
| Латинское (напр. `PoisonShot`) | `Справочники.Модификаторы.PoisonShot` |
| Русское (напр. `Антидот`) | `ПредопределенноеЗначение("Справочник.Модификаторы.Антидот")` |

**Пример 1: изменение шага уменьшения параметра** (в `УменьшитьПараметрыИгроков()`)

```bsl
Если ВидПараметра = Перечисления.ВидыПараметровИгроков.Яд
    И МодификаторАктивен(ПредопределенноеЗначение("Справочник.Модификаторы.Антидот")) Тогда
    НовоеЗначение = ЗначениеПараметра - 2;  // вместо -1
Иначе
    НовоеЗначение = ЗначениеПараметра - 1;
КонецЕсли;
```

**Пример 2: модификация урона от эффекта** (в `УменьшитьПараметрыПротивников()`)

```bsl
Если ВидПараметра = Перечисления.ВидыПараметровИгроков.Яд Тогда
    УронОтЯда = ЗначениеПараметра;
    Если МодификаторАктивен(ПредопределенноеЗначение("Справочник.Модификаторы.Токсикология")) Тогда
        УронОтЯда = УронОтЯда + 2;
    КонецЕсли;
    НанестиУронПротивнику(УронОтЯда);
КонецЕсли;
```

> **Важно:** проверка `МодификаторАктивен()` работает с таблицей `Модификаторы` на форме. Убедитесь, что процедура выполняется на сервере, где доступна эта таблица.

### Модификатор, срабатывающий при получении урона по здоровью (ЧумнойНоситель)

Паттерн «возвратный яд»: если модификатор активен, при прохождении урона по здоровью игрока (после щита) противник получает 1 Яд.

**Где:** процедура `НанестиУронИгроку(УронВход)` — в блоке `Если НовыйУрон > 0 Тогда` после всех модификаторов защиты (КожаныеПластины, Кольчуга), перед вычитанием здоровья.

```bsl
// Модификатор "ЧумнойНоситель": если урон прошёл по здоровью, противник получает 1 Яд
Если НовыйУрон > 0 И МодификаторАктивен(ПредопределенноеЗначение("Справочник.Модификаторы.ЧумнойНоситель")) Тогда
    УвеличитьПараметрПротивника(Перечисления.ВидыПараметровИгроков.Яд, 1);
КонецЕсли;
```

**Ключевые моменты:**
- Проверка `НовыйУрон > 0` гарантирует, что урон реально достиг здоровья (щит не поглотил, СтеклянныйЩит не сработал, КожаныеПластины/Кольчуга не обнулили)
- Добавляется **после** модификаторов, уменьшающих урон, но **до** вычитания из `ТекущееЗдоровье`
- Использует `УвеличитьПараметрПротивника()` — накопление параметра (не перезапись)

### Модификатор, срабатывающий при атаке игрока (ЯдовитыеКлинки)

Паттерн «ядовитые атаки»: если модификатор активен, при каждой атаке игрока противник дополнительно получает 1 Яд.

**Где:** процедура `УронПротивникуПоКоличеству(УронВход)` — после обработки всех модификаторов данной атаки (ЛетучаяМышь, ЦепкийКрюк), перед проверкой победы.

```bsl
// Применяем модификатор "ЯдовитыеКлинки": добавляет противнику 1 яда при атаке
Если МодификаторАктивен(ПредопределенноеЗначение("Справочник.Модификаторы.ЯдовитыеКлинки")) Тогда
    УвеличитьПараметрПротивника(Перечисления.ВидыПараметровИгроков.Яд, 1);
КонецЕсли;
```

**Ключевые моменты:**
- Добавляется **после** модификаторов урона (ЛетучаяМышь, ЦепкийКрюк), но **до** проверки победы
- Срабатывает при каждой атаке игрока, включая дополнительные атаки (например, от ПарныхКлинков)
- При активном ПарныеКлинки: рекурсивный вызов `АтакаИгрока()` → `УронПротивникуПоКоличеству()` → яд накладывается при каждой атаке

## Рефакторинг: замена прямых вызовов ПереместиКартуИзСтопки() на ДобратьВРукуКарты()

### Когда применять

Если в коде есть повторяющиеся циклы добора карт из `СтопкаДобора` в `РукаИгрока` через прямой вызов `ПереместиКартуИзСтопки()` — заменять на `ДобратьКартуВРуку()` (1 карта) или `ДобратьВРукуКарты(N)` (N карт).

### Шаги

1. **Сделать `ДобратьКартуВРуку()` и `ДобратьВРукуКарты()` серверными** — изменить директиву с `&НаКлиенте` на `&НаСервере`, чтобы они были доступны и из серверных процедур.

2. **Заменить все вызовы** по шаблону:

   **Добор 1 карты (с проверкой пустоты):**
   ```bsl
   // Было:
   Если СтопкаДобора.Количество() > 0 Тогда
       ПерваяКарта = СтопкаДобора[0];
       ИдентификаторКарты = ПерваяКарта.ИдентификаторКарты;
       ПереместиКартуИзСтопки(ИдентификаторКарты, "СтопкаДобора", "РукаИгрока", ЭтаФорма);
   КонецЕсли;
   
   // Стало:
   ДобратьКартуВРуку();
   ```

   **Добор N карт (цикл):**
   ```bsl
   // Было:
   Для Счетчик = 1 По N Цикл
       Если СтопкаДобора.Количество() > 0 Тогда
           ПерваяКарта = СтопкаДобора[0];
           ИдентификаторКарты = ПерваяКарта.ИдентификаторКарты;
           ПереместиКартуИзСтопки(ИдентификаторКарты, "СтопкаДобора", "РукаИгрока", ЭтаФорма);
       КонецЕсли;
   КонецЦикла;
   
   // Стало:
   ДобратьВРукуКарты(N);
   ```

   **Сложный случай (с перетасовкой сброса при пустой стопке):**
   ```bsl
   // Было:
   Если СтопкаДобора.Количество() > 0 Тогда
       ...
   Иначе
       ПеремешатьСтопкуДобора_Сервер();  // или ПеремешатьСбросВДобор()
       Если СтопкаДобора.Количество() > 0 Тогда
           ...
       КонецЕсли;
   КонецЕсли;
   
   // Стало:
   ДобратьКартуВРуку();  // автоперемешивание встроено
   ```

3. **Исключение: когда НЕ заменять.** Если после добора нужна ссылка на взятую карту (например, `ПерваяКарта.Карта.Стоимость`), оставить прямой вызов:

   ```bsl
   // НЕ заменять — нужна ссылка на карту для дальнейшей проверки:
   ПерваяКарта = СтопкаДобора[0];
   ИдентификаторКарты = ПерваяКарта.ИдентификаторКарты;
   ПереместиКартуИзСтопки(ИдентификаторКарты, "СтопкаДобора", "РукаИгрока", ЭтаФорма);
   Если ПерваяКарта.Карта.Стоимость = 0 Тогда
       ДобратьКартуВРуку();  // второй добор уже можно заменить
   КонецЕсли;
   ```

| `ДействиеКарты_Тройка()` | `&НаСервере` | `ДобратьВРукуКарты(3)` |
| `ДействиеКарты_Пятерка()` | `&НаСервере` | `ДобратьВРукуКарты(5)` |
| `ДействиеКарты_Передышка()` | `&НаКлиенте` | `ДобратьКартуВРуку()` |
| `ДействиеКарты_Замена()` | `&НаКлиенте` | `ДобратьВРукуКарты(2)` |
| `ДействиеКарты_Приют()` | `&НаКлиенте` | `ДобратьКартуВРуку()` |
| `НаградаСерии_Завершение()` | `&НаКлиенте` | `ДобратьВРукуКарты(Размер)` |

## Модификатор, срабатывающий в начале боя (ЗаводнаяПерчатка)

Паттерн «бесплатный розыгрыш»: если модификатор активен, при входе на этап с противником игрок бесплатно разыгрывает случайную карту из руки.

**Где:** процедура `ОбработатьВыбранныйВидЭтапа(ВыбранноеЗначение)` — блок `Если ВидЭтапа = ...Противник`, после других модификаторов (например, СтальныеНаручи), перед отображением задания.

```bsl
// Применяем модификатор "ЗаводнаяПерчатка": в начале боя игрок бесплатно разыгрывает 
// 1 случайную карту из руки
Если МодификаторАктивен(ПредопределенноеЗначение("Справочник.Модификаторы.ЗаводнаяПерчатка"))
    И РукаИгрока.Количество() > 0 Тогда
    Индекс = ГсчКлиент.СлучайноеЧисло(0, РукаИгрока.Количество() - 1);
    ВыбраннаяСтрока = РукаИгрока[Индекс];
    РазыгратьКарту(ВыбраннаяСтрока.ИдентификаторКарты);
КонецЕсли;
```

**Ключевые моменты:**
- Проверка `РукаИгрока.Количество() > 0` обязательна — рука может быть пуста
- Использует `ГсчКлиент` (не `Гсч`), т.к. `ОбработатьВыбранныйВидЭтапа` — клиентская (`&НаКлиенте`)
- Использует готовую процедуру `РазыгратьКарту(ИдентификаторКартыВход)` — выполняет действие карты, перемещает в сброс, обновляет состояние
- Размещать **после** применения стартовых модификаторов защиты (СтальныеНаручи), но **до** отображения первого задания (`ОтобразитьСледующееЗадание`)

### Доступные виды параметров

Перечисление `Перечисления.ВидыПараметровИгроков` содержит (для справки):
- `Яд` — наносит урон, равный значению, каждый ход до уменьшения
- `Горение` — наносит урон от горения каждый ход
- `Бомба` — взрывается при достижении 0
- `Щит`, `Сила`, `Неуклюжесть`, `Проклятие`, `БлокировкаЛечения`, `ЛедянойЩит`
- `СтеклянныйЩит` — хрупкая защита, добавляемая картами (например, карта СтеклянныйЩит даёт +1)
- и др.

## Шаг 5. Схема расхода ресурса (например, золота)

Использовать готовый метод `ЗаплатитьЗолото(Сумма)` — он сам проверяет наличие золота и списывает его:

```bsl
Если ЗаплатитьЗолото(Стоимость) Тогда
    // эффект карты
Иначе
    Сообщить("Недостаточно золота для карты " + ИмяКарты);
КонецЕсли;
```

Метод возвращает `Истина` при успешном списании и `Ложь`, если золота не хватило.

## Шаг 6. Карты без расхода ресурсов (получение / самоповреждение)

Если карта даёт ресурсы (щит, золото) без затрат — использовать `ПолучитьЗолото()`:

```bsl
УвеличитьЩитИгрока(5);
ПолучитьЗолото(5);
```

Если карта наносит урон самому себе в обмен на ресурс — порядок важен: сначала урон, потом начисление (при отрицательном здоровье эффект не сработает, но логика сохраняется):

```bsl
НанестиУронИгроку(10);  // сначала урон
ПолучитьЗолото(10);     // потом награда
```

## Шаг 8. Обработка структуры вознаграждения из оповещения формы

Когда форма открывается с `ОписаниеОповещения` и возвращает структуру с полями `Награда` (тип вознаграждения) и `Размер` (количество), используйте единое ветвление `Если..ИначеЕсли`:

```bsl
Процедура НаградаСерии_Завершение(ВыбранноеЗначение, ДополнительныеПараметры) Экспорт
    Если ТипЗнч(ВыбранноеЗначение) = Тип("Структура") Тогда
        Награда = ОбщегоНазначенияКлиентСервер.СвойствоСтруктуры(ВыбранноеЗначение, "Награда", Неопределено);
        Размер  = ОбщегоНазначенияКлиентСервер.СвойствоСтруктуры(ВыбранноеЗначение, "Размер", Неопределено);

        Если Награда = ПредопределенноеЗначение("Перечисление.ВидыПараметровИгроков.Золото") Тогда
            УвеличитьПараметрИгрока(ПредопределенноеЗначение("Перечисление.ВидыПараметровИгроков.Золото"), Размер);
        ИначеЕсли Награда = ПредопределенноеЗначение("Справочник.Карты.ПустаяСсылка") Тогда
            ДобратьВРукуКарты(Размер);
        ИначеЕсли Награда = ПредопределенноеЗначение("Перечисление.ВариантыДействийПротивников.Атака") Тогда
            УронПротивникуПоКоличеству(Размер);
        ИначеЕсли Награда = ПредопределенноеЗначение("Перечисление.ВидыПараметровИгроков.Энергия") Тогда
            УвеличитьПараметрИгрока(ПредопределенноеЗначение("Перечисление.ВидыПараметровИгроков.Энергия"), Размер);
        Иначе
            Сообщить("Неизвестный тип награды: " + Награда);
        КонецЕсли;

        ОтобразитьСледующееЗадание(Сборник);
    Иначе
        // Неизвестный формат ответа — ничего не делаем
    КонецЕсли;
КонецПроцедуры
```

### Описание типов наград

| Тип (`Награда`) | Механика | Действие |
|---|---|---|
| `ВидыПараметровИгроков.Золото` | Начислить золото | `УвеличитьПараметрИгрока(ВидПараметраЗолото, Размер)` |
| `Справочник.Карты.ПустаяСсылка` | Добор карт | `ДобратьВРукуКарты(Размер)` |
| `ВариантыДействийПротивников.Атака` | Атака противника | `УронПротивникуПоКоличеству(Размер)` |
| `ВидыПараметровИгроков.Энергия` | Восполнить энергию | `УвеличитьПараметрИгрока(ВидПараметраЭнергия, Размер)` |

> **Ключевые моменты:**
> - Используйте `ОбщегоНазначенияКлиентСервер.СвойствоСтруктуры(Структура, "Ключ", ЗначениеПоУмолчанию)` — безопасное чтение поля со значением по умолчанию.
> - Всегда проверяйте `ТипЗнч(ВыбранноеЗначение) = Тип("Структура")` перед разбором — значение может быть `Неопределено` при закрытии формы без выбора.
> - `СвойствоСтруктуры()` с `Неопределено` в качестве значения по умолчанию — если ключ отсутствует, тип награды будет `Неопределено`, и сработает ветка `Иначе`.
> - Ветка `Иначе` обязательна — сообщает о неизвестном типе награды на случай, если в будущем добавятся новые типы.

## Шаг 9. Чтение параметров из таблиц формы на клиенте

На клиенте `ТаблицаЗначений` недоступна, но реквизиты формы типа `ValueTable` (например, `ПараметрыИгрока`, `ПараметрыПротивника`) — это клиентские объекты `ТаблицаЗначений`, с которыми можно работать напрямую.

### Поиск строки по значению колонки

```bsl
&НаКлиенте
Процедура ПримерЧтенияПараметра()
    ВидЗдоровья = ПредопределенноеЗначение("Перечисление.ВидыПараметровИгроков.ТекущееЗдоровье");
    // Поиск строки
    СтруктураПоиска = Новый Структура;
    СтруктураПоиска.Вставить("ВидПараметра", ВидЗдоровья);
    НайденныеСтроки = ПараметрыПротивника.НайтиСтроки(СтруктураПоиска);
    // Чтение найденного значения
    Если НайденныеСтроки.Количество() > 0 Тогда
        ЗдоровьеПротивника = НайденныеСтроки[0].ЗначениеПараметра;
    Иначе
        ЗдоровьеПротивника = 0;
    КонецЕсли;
КонецПроцедуры
```

**Важно:** 
- `ПараметрыПротивника` и `ПараметрыИгрока` доступны на клиенте как реквизиты формы
- Для поиска используйте `НайтиСтроки(СтруктураПоиска)` — возвращает массив найденных строк
- Тернарные операторы `?(..., ..., ...)` не использовать — только классические `Если...Иначе`

### Полный пример: мгновенное убийство противника (карта ЧитКод)

```bsl
&НаКлиенте
Процедура ДействиеКарты_ЧитКод(НайденнаяКарта, ИдентификаторКартыВход)
    // Инициализация.
    ВидЗдоровья = ПредопределенноеЗначение("Перечисление.ВидыПараметровИгроков.ТекущееЗдоровье");
    ВидЩита = ПредопределенноеЗначение("Перечисление.ВидыПараметровИгроков.Щит");
    // Поиск здоровья противника.
    СтруктураПоискаЗдоровье = Новый Структура;
    СтруктураПоискаЗдоровье.Вставить("ВидПараметра", ВидЗдоровья);
    НайденныеСтрокиЗдоровье = ПараметрыПротивника.НайтиСтроки(СтруктураПоискаЗдоровье);
    Если НайденныеСтрокиЗдоровье.Количество() > 0 Тогда
        ЗдоровьеПротивника = НайденныеСтрокиЗдоровье[0].ЗначениеПараметра;
    Иначе
        ЗдоровьеПротивника = 0;
    КонецЕсли;
    // Поиск щита противника.
    СтруктураПоискаЩит = Новый Структура;
    СтруктураПоискаЩит.Вставить("ВидПараметра", ВидЩита);
    НайденныеСтрокиЩит = ПараметрыПротивника.НайтиСтроки(СтруктураПоискаЩит);
    Если НайденныеСтрокиЩит.Количество() > 0 Тогда
        ЩитПротивника = НайденныеСтрокиЩит[0].ЗначениеПараметра;
    Иначе
        ЩитПротивника = 0;
    КонецЕсли;
    // Нанесение суммарного урона = здоровье + щит.
    УронКарты = ЗдоровьеПротивника + ЩитПротивника;
    УронПротивникуПоКоличеству(УронКарты);
КонецПроцедуры
```

## Шаг 10. Модификатор избыточного лечения → щит (КровянойКлапан)

Паттерн «overheal conversion»: если модификатор активен, то при восстановлении здоровья, превышающем максимум, излишек превращается в щит.

**Где:** процедура `ВосстановитьЗдоровьеИгрока(КоличествоВосстановления)` — после расчёта `НовоеЗдоровье = Мин(...)`, перед `УстановитьПараметрИгрока`.

```bsl
НовоеЗдоровье = Мин(ТекущееЗдоровье + ФактическоеВосстановление, МаксимальноеЗдоровье);
// Применяем модификатор "КровянойКлапан": избыточное лечение обращается в щит.
Если МодификаторАктивен(Справочники.Модификаторы.КровянойКлапан) Тогда
    ИзбыточноеЛечение = (ТекущееЗдоровье + ФактическоеВосстановление) - МаксимальноеЗдоровье;
    Если ИзбыточноеЛечение > 0 Тогда
        УвеличитьЩитИгрока(ИзбыточноеЛечение);
    Иначе
        // Нет избыточного лечения.
    КонецЕсли;
Иначе
    // Модификатор не активен.
КонецЕсли;
УстановитьПараметрИгрока(ВидПараметраТекущееЗдоровье, НовоеЗдоровье);
```

**Ключевые моменты:**
- Проверка `ИзбыточноеЛечение > 0` предотвращает конвертацию нулевого/отрицательного избытка
- `УвеличитьЩитИгрока()` корректно учитывает другие модификаторы щита (например, ЛатныйНаплечник)
- Код вставляется **до** вызова `УстановитьПараметрИгрока`, чтобы здоровье уже было зафиксировано на максимуме
- Ветки `Иначе` с комментарием обязательны по правилам проекта (AI_RULES.md)

## Шаг 11. Мета-карта: разыгрывание других карт из руки (Зазеркалье)

Паттерн «meta-card»: карта, которая находит в руке другие карты по критерию и разыгрывает их. Пример: **Зазеркалье** — «разыграть все карты в руке стоимостью 0 дважды».

### Ключевые принципы

1. **Снапшот руки до розыгрыша** — собрать подходящие карты в массив, затем итерировать. Новые карты, добавленные эффектами уже разыгранных карт, не должны попадать в цикл.
2. **Исключить себя** — проверка `ТекСтрока.ИдентификаторКарты <> ИдентификаторКартыВход`, чтобы карта не пыталась разыграть саму себя.
3. **Два розыгрыша на каждую карту**:
   - Первый: `ДействиеВыбраннойКарты(Карта, Идентификатор)` — применяет эффект, карта остаётся в руке
   - Второй: `РазыгратьКарту(Идентификатор)` — применяет эффект повторно + перемещает в сброс + обновляет UI (`ОтобразитьСостояниеИгрока`, `ОтобразитьСостояниеПротивника`)

### Почему именно так

Нельзя вызвать `РазыгратьКарту` дважды: после первого вызова карта уже в сбросе, и второй вызов выдаст «Карта не найдена». Поэтому первый розыгрыш делается напрямую через `ДействиеВыбраннойКарты`, а второй через `РазыгратьКарту`, которая заодно корректно завершает жизненный цикл карты.

### Процедура: `&НаКлиенте`

```bsl
&НаКлиенте
Процедура ДействиеКарты_Зазеркалье(НайденнаяКарта, ИдентификаторКартыВход)
    // Собираем идентификаторы всех карт в руке стоимостью 0 (кроме самой разыгрываемой)
    МассивКартДляРозыгрыша = Новый Массив;
    Для Каждого ТекСтрока Из РукаИгрока Цикл
        Если ТекСтрока.ИдентификаторКарты <> ИдентификаторКартыВход
            И ТекСтрока.Карта.Стоимость = 0 Тогда
            СтруктураКарты = Новый Структура;
            СтруктураКарты.Вставить("Карта", ТекСтрока.Карта);
            СтруктураКарты.Вставить("ИдентификаторКарты", ТекСтрока.ИдентификаторКарты);
            МассивКартДляРозыгрыша.Добавить(СтруктураКарты);
        КонецЕсли;
    КонецЦикла;
    // Разыгрываем каждую найденную карту дважды
    Для Каждого ТекСтруктура Из МассивКартДляРозыгрыша Цикл
        // Первый розыгрыш: применяем эффект (карта остаётся в руке)
        ДействиеВыбраннойКарты(ТекСтруктура.Карта, ТекСтруктура.ИдентификаторКарты);
        // Второй розыгрыш + сброс + обновление UI через РазыгратьКарту
        РазыгратьКарту(ТекСтруктура.ИдентификаторКарты);
    КонецЦикла;
КонецПроцедуры
```

### Адаптация под другой критерий

Замените условие `ТекСтрока.Карта.Стоимость = 0` на нужное:
- **Все карты конкретного типа**: `ТекСтрока.Карта = ПредопределенноеЗначение("Справочник.Карты.ИмяКарты")`
- **Все карты** (кроме себя): убрать второе условие, оставить только `<> ИдентификаторКартыВход`
- **N случайных карт**: вместо сбора всех — выбрать N случайных индексов через `ГсчКлиент`

### Важные ограничения

- **Директива `&НаКлиенте` обязательна** — `ДействиеВыбраннойКарты` и `РазыгратьКарту` обе клиентские
- **Не использовать на сервере** — серверный код не может вызывать клиентский `ДействиеВыбраннойКарты` напрямую
- **Рекурсия**: если разыгрываемая карта добавляет в руку новые карты, подходящие под критерий, они **не** будут разыграны (снапшот собран до цикла)
- **Пустая рука**: если подходящих карт нет, массив пуст — цикл просто не выполнится
