# QWEN.md — Veritas: мобильный тренажёр для запоминания

## Обзор проекта

**Veritas** — мобильное приложение-тренажёр на платформе **1С:Предприятие 8.3.26+**, работающее в режиме управляемых форм (`ManagedApplication`) и предназначенное только для мобильной платформы (`MobilePlatformApplication`). Приложение реализует методику интервального повторения и адаптивного обучения через игровые механики: викторины, восхождения (roguelike-режим), противников, модификаторы, достижения и систему уровней.

- **Версия конфигурации:** 1.5.2
- **Вендор:** Gigabyte (blog.livegig.ru)
- **Интерфейс:** Такси (Taxi)
- **Режим совместимости:** 8.3.26
- **Язык скриптов:** Русский (BSL)
- **Режим модальности:** DontUse

## Архитектура и ключевые компоненты

### Режимы игры

1. **Викторина** (`РежимыВикторины.Викторина`) — классический режим: вопрос → N вариантов ответа (4 или 6, зависит от сложности сборника).
2. **Восхождение** (`РежимыВикторины.Восхождение`) — roguelike-режим: этапы, противники, карты, модификаторы, отдых, торговец.

### Основной игровой движок: `DataProcessors.ТестированиеЗнаний`

Это центральный компонент. Его **ManagerModule** (`Ext/ManagerModule.bsl`) содержит общие функции:
- `НайтиСлучайноеЗадание()` — выбор случайного задания из сборника
- `ПолучитьСлучайныйСборник()` — выбор случайного сборника
- `ЗаписатьОтветВРезультаты()` — запись результата ответа в регистр
- `ПолучитьМассивОтветовКУдалениюПоКоличеству()` — фильтрация вариантов ответов
- `ЭтоЭтапЭлитнойСложности()` — проверка элитных этапов

**Главная форма** (`Forms/Форма/Ext/Form/Module.bsl`, ~5300 строк) — ядро UI и игровой логики:
- Генерация вариантов ответов (`ПолучитьВариантыОтветов`)
- Обработка выбора ответа, анимации, таймеры
- Управление HUD (здоровье, энергия, монеты)
- Механики восхождения (этапы, противники, карты)

**12 подформ** для отдельных экранов:
- `ФормаДействияПротивника` — экран атаки противника
- `ФормаИнформацииОБое` — информация о текущем бое
- `ФормаТорговца` — экран торговца (покупка карт/модификаторов)
- `ФормаОтдыха` — экран отдыха между этапами
- `ФормаПолученияНаград` / `ФормаНаградаСерии` — экраны наград
- `ФормаНастроекВикторины` / `ФормаНачальныхНастроек` — настройки
- `ФормаВыбораЭтапа` — выбор этапа восхождения
- `ФормаПораженияВосхождение` / `ФормаУспешноеВосхождение` — результаты
- `ФормаПросмотраКартинки` — просмотр картинок заданий

### Общие модули

| Модуль | Назначение |
|--------|-----------|
| `ОбщегоНазначенияКлиентСервер` | Утилиты: `СвернутьМассив()`, `СмешатьМассив()`, `СвойствоСтруктуры()`, `ДополнитьМассив()`, `ПолучитьТекстПодсказки()`, `ВернутьУровеньПоОпыту()` |
| `ОбщегоНазначенияКлиент` | Клиентские утилиты |
| `ОбщегоНазначенияВызовСервера` | Серверные вызовы (например, `ОпределитьКоличествоУстановленныхСборников()`) |
| `ДостиженияКлиентСервер` | Проверка условий достижений |
| `ОбменССайтом` | Обмен данными с внешним сайтом |

### Регистры сведений

- **ДанныеПользователей** — опыт, монеты, уровень игрока
- **РезультатыТестированияЗнаний** — история ответов (верно/неверно) с периодичностью
- **ДостиженияПользователей** — разблокированные достижения
- **СохраненияВосхождения** — сохранённые состояния roguelike-режима
- **ЭтапыВосхождения** — конфигурация этапов (номера, сложность, типы)

## Стандарты кодирования (AI_RULES.md)

Все изменения должны строго соблюдать правила из `AI_RULES.md`. Ключевые требования:

### Именование
- **Переменные:** CamelCase/PascalCase, префикс `Тек` для итераторов, суффикс `Вход` для входных параметров (неизменяемых), суффикс `Изм` для модифицируемых параметров
- **Булевы:** начинаются с `Это`, `Есть` или `Разрешено`
- **Процедуры/Функции:** PascalCase, формат `Глагол` + `Существительное`
- **Результат функций:** переменная `РезультатФункции` перед `Возврат`

### Директивы компиляции
- `&НаКлиенте` — UI и события формы
- `&НаСервере` — серверный контекст
- `&НаСервереБезКонтекста` — предпочтительный для бизнес-логики и запросов
- `&НаКлиентеНаСервереБезКонтекста` — универсальные методы, форма передаётся параметром

### Форматирование
- **Запрещены пустые строки внутри тела метода** — только между методами
- Логические блоки разделяются комментариями-заголовками (`// Инициализация.`)
- Одна строка — одно действие (запрет составных выражений)
- Каждый `Если` должен иметь блок `Иначе` (с комментарием, если пустой)
- Инициализация всех констант и перечислений в начале метода с комментарием `// Инициализация.`
- Метод помещается на 1-2 экрана

### Запросы
- Пакетная обработка, запрет запросов в цикле
- Параметры (`&Параметр`) вместо подстановки значений в текст
- Форматирование через `|`
- `РАЗРЕШЕННЫЕ РАЗЛИЧНЫЕ` где нужно

### Интерфейс
- Асинхронные диалоги через `ОписаниеОповещения`
- Локализация пользовательских строк через `НСтр("ru = '...'")`
- Централизованное управление доступностью: `УправлениеДоступностью(Форма)`
- Предопределённые значения через `ПредопределенноеЗначение("...")`

### Данные
- `ЗначениеЗаполнено()` для проверки ссылок, дат, строк, чисел
- Сравнение с `Неопределено` для структур, массивов
- Явное создание структур через `Вставить` вместо конструктора со строкой ключей

## Сборка и запуск

Проект 1С не имеет традиционных команд сборки. Рабочий процесс:

1. **Загрузка конфигурации:** конфигурация загружается в среду 1С:Предприятие через Конфигуратор (`Configuration.xml` как точка входа).
2. **Обновление базы данных:** после изменений метаданных требуется обновление конфигурации БД (F7 в Конфигураторе).
3. **Тестирование:** запуск в режиме «1С:Предприятие» (управляемое приложение). Для мобильной платформы — публикация через «Мобильное приложение» и запуск на мобильном клиенте 1С.
4. **Версионирование:** версия указывается в `Configuration.xml` (Properties/Version).

Расширение файлов `.bsl` — исходный код на встроенном языке 1С. Файлы `.xml` в директориях `Ext/` описывают метаданные (реквизиты, формы, команды).

## Заметки по проекту

- **Только мобильная платформа:** конфигурация не предназначена для десктопного/thin-клиента.
- **Оффлайн-режим:** приложение работает без подключения к интернету.
- **Единственная включённая мобильная функциональность:** `AudioPlaybackAndVibration`.
- **Игровые механики:** система уровней (1-45), достижения, монеты, противники с действиями, карты и модификаторы как коллекционируемые/расходуемые предметы.
- **Справочник `Задания`** подчинён `Сборникам` (реквизит `Владелец`). Каждое задание имеет табличную часть `ОтветыЗаданий` с вариантами ответов и флагом правильности.
